# Оформление бота CitySignal

Материалы и шаги, чтобы «упаковать» публичного бота: картинка к описанию,
приветствие на `/start`, команды, About, кнопка меню.

## Что уже в коде (этой сессией)

- `services/curator/app/routers/bot.py` — webhook `/tg/webhook`: на `/start`
  отвечает брендовым приветствием с inline-кнопкой web_app «Открыть CitySignal»;
  плюс `/help` и `/feedback`. Без отдельного процесса — висит на уже
  развёрнутом curator. Токен берёт из `TELEGRAM_BOT_TOKEN` (или `CS_BOT_TOKEN`).
- `services/curator/app/setup_bot.py` — идемпотентный скрипт: одним прогоном
  ставит webhook + команды + описание + короткое описание + кнопку меню.

## Что делает ТОЛЬКО человек

### 1. Картинка к описанию (единственное, чего нет в Bot API)

Файл: **`bot-setup/description-640x360.png`** (640×360, готов).

@BotFather → `/mybots` → выбрать бота → **Edit Bot → Edit Description Picture**
→ прикрепить `description-640x360.png`. Появится над блоком «Что умеет этот бот?».

### 2. Задеплоить `/start` и прогнать setup_bot

На сервере (curator уже знает токен бота через `TELEGRAM_BOT_TOKEN`):

```bash
ssh root@45.144.52.40
cd /root/react_native_sf && git pull --ff-only
docker compose build curator && docker compose up -d curator
# один раз: ставит webhook, команды, описание, кнопку меню
docker compose exec curator python -m app.setup_bot
# проверка
docker compose exec curator python -m app.setup_bot --info
```

После этого `/start` отвечает приветствием с кнопкой, а `/` показывает команды.
