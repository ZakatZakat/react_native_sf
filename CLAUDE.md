# react_native_sf (CitySignal) — заметки для агентов

Над проектом параллельно работает **несколько сессий/агентов в одном репозитории**.
Правила ниже — чтобы сессии не затирали друг друга (ни в git, ни на проде).

## ⚠️ Изоляция сессий — коммить и деплой ТОЛЬКО своё

Симптом, которого избегаем: одна сессия закоммитила/выкатила чужой незакоммиченный
WIP, работа другой сессии затёрлась или уехала на прод недоделанной.

**Перед задачей — ветвись от свежего master:**
```
git fetch origin
git checkout -B <своя-ветка> origin/master
```
Если рабочее дерево уже занято чужим незакоммиченным WIP (частый случай — тут
несколько чатов) — делай задачу в **отдельном git worktree**, чужие правки не трогая:
```
git worktree add ../csw-<задача> -b <своя-ветка> origin/master
cd ../csw-<задача>      # изолированная копия дерева
```

**Коммить ТОЛЬКО свои файлы — никогда `git add -A` / `git add .` в общем дереве:**
```
git add <конкретные пути, которые правил именно ты>
git commit -m "…"
```
Перед коммитом сверься `git status`: всё, что ты НЕ трогал (чужой WIP), должно
остаться незастейдженным. Если сомневаешься, чьё изменение — не коммить его.

## ⚠️ Деплой на прод — ТОЛЬКО через `scripts/deploy.sh`

**НЕ делать руками `scp` / `rsync` / `docker build` на прод-сервер** и **не** править
файлы на боксе напрямую. Из-за этого дерево на боксе обрастает незакоммиченными
правками разных сессий (Франкенштейн), а `git pull` на боксе перестаёт быть
чистым.

Инвариант: **на проде всегда ровно один коммит из `origin/master`.** Поток:
```
своя ветка → влить в master → git push origin master → ./scripts/deploy.sh
```
- `./scripts/deploy.sh` — катит **только** то, что уже в `origin/master`, под
  серверным локом (две сессии одновременно не выкатят), обновляет бокс через
  `git fetch` + `reset --hard <sha>` (untracked-файлы и `.env` сохраняются),
  затем `docker compose build/up` нужного сервиса.
- `./scripts/deploy.sh --service curator` — по умолчанию катится `app`; так
  выбираешь бэкенд (или `--service "app curator"`).
- `./scripts/deploy.sh --check` — показывает, **чей коммит сейчас на проде**
  (`git rev-parse` на боксе). Перед выкатом сверяйся, что не откатываешь чужое.

Если скрипту чего-то не хватает — **допиливаем скрипт**, а не катим в обход.
Механика сервисов, БД, кэша Telegram — в **DEPLOY.md**.

## Стиль коммитов (важно)

- Сообщения — **на русском**, прошедшее время / страдательный залог
  («Добавлено», «Исправлено», «Убрано»). Без conventional-commit префиксов.
- **Без** trailer'ов `Co-Authored-By` и без пометок про генерацию.
- `gh pr create` / `gh pr merge` — **только по явной команде владельца**.

## Чего НЕ трогать на сервере

На `45.144.52.40` крутится много чужих проектов (avia-platform, sok,
digital-assistant, analytics-platform, VPN xray/x-ui и т.п.). Работать только с
compose-проектом `react_native_sf`. Ничего лишнего не останавливать/не удалять,
серверный `.env` не перезаписывать.
